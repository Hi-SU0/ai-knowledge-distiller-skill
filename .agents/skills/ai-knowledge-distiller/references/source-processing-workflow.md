# Source Processing Workflow

## Information acquisition priority

Use the highest-quality accessible material and record the actual material obtained:

1. Official transcript, publisher-provided text, original article, original paper, or official documentation.
2. Source metadata: title, description, chapters, pinned comments, release notes, and linked resources.
3. Automatically generated transcript, only after checking names, terminology, punctuation, and obvious recognition errors.
4. Slides, code, charts, webpages, demonstrations, and other key visual evidence.
5. Matching creator articles, official repositories, standards, or primary documentation.
6. Third-party notes only as attributed assistance, never as the source's original words.

If the primary content is inaccessible, do not infer it from the title. Record what is available, lower completeness, identify missing evidence, and complete only the reliable portion.

## Step 1: Establish a source record

Record title, creator or organization, source type, platform, original link, identifier, publication date, duration or page count when applicable, processed date, material types, transcript completeness, visual analysis coverage, and information completeness (`高`, `中`, or `低`).

## Step 2: Reconstruct the source structure

Identify:

- the core question;
- target audience;
- prerequisites;
- chapters or logical sections;
- argument or teaching sequence;
- final conclusion.

Rebuild the knowledge structure rather than shortening each transcript sentence.

## Step 3: Extract useful content

Produce:

- a one-sentence summary of no more than 100 Chinese characters when writing Chinese;
- a 300–500 Chinese-character overview when the source supports it;
- core question;
- three to eight core conclusions;
- key concepts;
- argument chain;
- cases, analogies, demonstrations, and visual evidence;
- practical methods;
- likely misunderstandings;
- questions the source leaves open.

Retain useful timestamps, pages, or section anchors. Quote only short, essential passages and respect copyright limits.

## Step 4: Distill knowledge classes

Classify extracted content as:

- **A. Atomic knowledge**: reusable concepts that can stand alone and link to other notes.
- **B. Source context**: material meaningful primarily inside the current argument.
- **C. Creator viewpoint**: personal judgment, experience, preference, or prediction that must remain attributed.

Create or update separate concept notes for class A. Keep B in the source note. Keep C visibly labeled and traceable.

## Step 5: Link across the vault

Inspect existing notes for:

- identical, broader, narrower, and prerequisite concepts;
- related tools, cases, and projects;
- similar or opposing viewpoints;
- later advanced material;
- changes in a creator's view over time;
- agreements and conflicts across creators.

Update global concept notes and topic summaries. Create a comparison note when differences are substantial enough to affect definitions, technical choices, applicability, or learning decisions.

## Step 6: Verify facts

Prioritize verification of:

- product details that may have changed;
- software versions, interfaces, and API behavior;
- model capabilities, prices, and context lengths;
- company, product, and personnel information;
- industry trends and predictions;
- disputed technical judgments;
- unsourced statistics.

Use primary sources when possible. Label each important statement as `来源明确陈述`, `合理推导`, `创作者个人观点`, `外部补充`, `暂未验证`, or `可能已经过时`. External verification belongs in a separate section with its own source.

## Step 7: Write and update indexes

1. Create or update the source note.
2. Create or update global atomic concepts after deduplication.
3. Update relevant topic summaries and viewpoint comparisons.
4. Update the knowledge map, topic index, and creator/source index.
5. Update the processing or maintenance record.
6. Report created files, modified files, new concepts, cross-source links, unresolved information, and the next recommended source.

## Batch discipline

- Start with a small representative sample, normally three sources.
- Expand only after the user validates structure and detail level.
- After every five sources, merge duplicates, repair links, refresh maps and summaries, check conflicts, list gaps, and reassess the recommended learning order.
